‚ÑπÔ∏è
ACID Properties:
  • Atomicity: Transaksi dilakukan seluruhnya atau tidak sama sekali
  • Consistency: Data dalam basis data tetap konsisten

1

2

  • Di browser: http://localhost/phpmyadmin
  • Buat database: rumahsakit (atau gunakan dari T3-1)
  • Pastikan tabel menggunakan InnoDB (mendukung transaksi)
3

Verifikasi Storage Engine (PENTING!)

  • Di phpMyAdmin, pilih database rumahsakit
  • Lihat tabel-tabel yang ada ‚Üí Kolom "Type" harus menunjukkan InnoDB
  • Jika masih MyISAM, ubah: Klik tabel ‚Üí Operations ‚Üí Change table type to InnoDB

üí° Catatan Penting

  • Dalam praktikum ini, semua command SQL dimasukkan melalui menu SQL di phpMyAdmin

üí≥ Bagian 13: Praktikum Transaksi - COMMIT

Pemahaman COMMIT

COMMIT: Perintah yang menandakan transaksi telah selesai dan data berhasil disimpan ke database.

Langkah-langkah Praktikum COMMIT

1

Buat Stored Procedure Transaksi1 (TANPA COMMIT)

Di phpMyAdmin, pilih database rumahsakit ‚Üí menu Routines ‚Üí Add Routine

Isikan dengan konfigurasi berikut:

Field Nilai
Routine name transaksi1
Routine type PROCEDURE
Definition Lihat code di bawah

Pada field Definition, masukkan SQL code:

BEGIN INSERT INTO pasien (idpasien, namapasien, alamatpasien, jeniskelamin) VALUES (6, 'Tito', 'Jl Pakis Haji No 25', 1); END

‚ö†Ô∏è PERHATIAN: Tidak ada COMMIT di sini! Ini akan ditunjukkan nanti.

2

Jalankan Transaksi1

  • Di phpMyAdmin, pilih database rumahsakit
  • Buka tab SQL
  • Ketik dan jalankan:
CALL transaksi1;
  • Sistem akan menampilkan bahwa data dengan idpasien 6 telah diinsert
3

Cek Tabel Pasien

  • Klik tabel pasien di sidebar
  • Hasil yang seharusnya terjadi: Data idpasien 6 TIDAK ada!
  • Mengapa? Karena transaksi belum di-COMMIT, jadi perubahan data tidak tersimpan permanen
4

Buat Stored Procedure Transaksi2 (DENGAN COMMIT)

Buat routine baru dengan nama transaksi2, Definition:

BEGIN INSERT INTO pasien (idpasien, namapasien, alamatpasien, jeniskelamin) VALUES (6, 'Tito', 'Jl Pakis Haji No 25', 1); COMMIT; END
5

Jalankan Transaksi2

CALL transaksi2;
6

Cek Tabel Pasien Lagi

  • Klik tabel pasien
  • Hasil yang seharusnya terjadi: Data idpasien 6 SEKARANG ADA!
  • Mengapa? Karena transaksi sudah di-COMMIT, perubahan data tersimpan permanen di database
‚úÖ KESIMPULAN COMMIT: Jika transaksi belum COMMIT, data tidak akan tersimpan ke database. COMMIT adalah perintah untuk mengkonfirmasi dan menyimpan perubahan data secara permanen.

‚Ü©Ô∏è Bagian 14: Praktikum Transaksi - ROLLBACK

Pemahaman ROLLBACK

ROLLBACK: Perintah untuk membatalkan transaksi dan mengembalikan data ke kondisi sebelum transaksi dilakukan.

Langkah-langkah Praktikum ROLLBACK

1

Buat Stored Procedure Transaksi3 (UPDATE DENGAN ROLLBACK)

Kita akan coba update nama pasien dengan id 3 dari 'Popi' menjadi 'Danar', tapi akan di-ROLLBACK:

BEGIN UPDATE pasien SET namapasien = 'Danar' WHERE idpasien = 3; ROLLBACK; SELECT * FROM pasien WHERE idpasien = 3; END
2

Jalankan Transaksi3

CALL transaksi3;
3

Hasil yang Seharusnya Terjadi

  • SELECT akan menampilkan data pasien id 3 dengan nama masih 'Popi' (belum berubah)
  • Mengapa? Karena UPDATE di-ROLLBACK sebelum di-COMMIT
  • ROLLBACK membatalkan semua perubahan dalam transaksi
4

Buat Stored Procedure Transaksi4 (INSERT + UPDATE + ROLLBACK PARTIAL)

Kali ini: INSERT data baru, ROLLBACK, terus UPDATE data lain tanpa ROLLBACK:

BEGIN INSERT INTO pasien (idpasien, namapasien, alamatpasien, jeniskelamin) VALUES (8, 'Budi', 'Jl Sudirman', 1); ROLLBACK; UPDATE pasien SET namapasien = 'Septin' WHERE idpasien = 3; COMMIT; END
5

Jalankan Transaksi4

CALL transaksi4;
6

Hasil yang Seharusnya Terjadi

  • INSERT dibatalkan: Tidak ada pasien id 8 di tabel (karena ROLLBACK)
  • UPDATE tersimpan: Pasien id 3 sekarang bernama 'Septin' (karena ada COMMIT setelahnya)
  • ROLLBACK hanya membatalkan command sebelumnya, bukan seluruh transaksi
‚úÖ KESIMPULAN ROLLBACK: ROLLBACK membatalkan transaksi dan mengembalikan data ke kondisi sebelumnya. Hanya perubahan yang sudah di-COMMIT yang akan tersimpan permanen.

üè∑Ô∏è Bagian 15: Praktikum Transaksi - SAVEPOINT / CHECKPOINT

Pemahaman SAVEPOINT

SAVEPOINT (Checkpoint): Menandai titik tertentu dalam transaksi. Kita bisa ROLLBACK ke savepoint tertentu, bukan ke awal transaksi.

Langkah-langkah Praktikum SAVEPOINT

1

Buat Stored Procedure Transaksi5 (DENGAN SAVEPOINT)

Kita akan INSERT 3 data, tapi yang ke-2 akan di-ROLLBACK ke savepoint point1:

BEGIN INSERT INTO pasien (idpasien, namapasien, alamatpasien, jeniskelamin) VALUES (7, 'Ari', 'Jl Ahmad Yani', 1); SAVEPOINT point1; INSERT INTO pasien (idpasien, namapasien, alamatpasien, jeniskelamin) VALUES (8, 'Budi', 'Jl Gatot Subroto', 1); ROLLBACK TO SAVEPOINT point1; INSERT INTO pasien (idpasien, namapasien, alamatpasien, jeniskelamin) VALUES (9, 'Citra', 'Jl Merdeka', 2); COMMIT; END
2

Jalankan Transaksi5

CALL transaksi5;
3

Hasil yang Seharusnya Terjadi

  • Pasien id 7 (Ari): ADA (INSERT pertama sebelum savepoint)
  • Pasien id 8 (Budi): TIDAK ADA (INSERT ke-2 di-rollback ke point1)
  • Pasien id 9 (Citra): ADA (INSERT ke-3 setelah rollback)

SAVEPOINT memungkinkan kontrol granular: rollback ke titik tertentu tanpa membatalkan seluruh transaksi!

‚úÖ KESIMPULAN SAVEPOINT: SAVEPOINT adalah checkpoint dalam transaksi. Gunakan ROLLBACK TO SAVEPOINT untuk membatalkan ke titik tertentu, bukan ke awal transaksi.

‚ö° Bagian 16: Praktikum TRIGGER Database

Pemahaman TRIGGER

TRIGGER: Perintah otomatis yang dijalankan sebelum atau sesudah event tertentu (INSERT, UPDATE, DELETE) pada tabel.

Jenis Trigger Penjelasan
BEFORE INSERT Dijalankan SEBELUM data diinsert
AFTER INSERT Dijalankan SESUDAH data diinsert
BEFORE UPDATE Dijalankan SEBELUM data diupdate
AFTER UPDATE Dijalankan SESUDAH data diupdate
BEFORE DELETE Dijalankan SEBELUM data dihapus
AFTER DELETE Dijalankan SESUDAH data dihapus

Persiapan: Membuat Tabel Obat & Transaksi Obat

1

Buat Tabel OBAT

Di phpMyAdmin, SQL tab, copy-paste:

CREATE TABLE obat ( idobat INT(6) AUTO_INCREMENT PRIMARY KEY, kodeobat VARCHAR(5), namaobat VARCHAR(30), harga INT(10), stok INT(5) );
2

Buat Tabel TRANSAKSI_OBAT

CREATE TABLE transaksiobat ( idtransaksi INT(3) AUTO_INCREMENT PRIMARY KEY, idpasien INT(3), idobat INT(3), jumlah INT(10), totalharga INT(10), FOREIGN KEY (idpasien) REFERENCES pasien(idpasien), FOREIGN KEY (idobat) REFERENCES obat(idobat) );
3

Insert Data Obat (Minimal 3 data)

INSERT INTO obat (kodeobat, namaobat, harga, stok) VALUES ('A001', 'Amoxillin', 50, 20), ('A002', 'Paracetamol', 10, 100), ('A003', 'Antibiotik', 75, 15);

Praktikum TRIGGER: Otomatis Kurangi Stok Obat

üìù
Kasus: Saat pasien membeli obat dan data diinsert ke tabel transaksiobat, stok obat di tabel obat harus otomatis berkurang sesuai jumlah yang dibeli.
1

Buat TRIGGER tgjual (After Insert)

Di phpMyAdmin, pilih database ‚Üí tab SQL, copy-paste:

DELIMITER $$ CREATE TRIGGER tgjual AFTER INSERT ON transaksiobat FOR EACH ROW BEGIN UPDATE obat SET stok = stok - NEW.jumlah WHERE idobat = NEW.idobat; END$$ DELIMITER ;

Penjelasan:

  • tgjual: Nama trigger
  • AFTER INSERT ON transaksiobat: Dijalankan sesudah insert data ke transaksiobat
  • NEW.jumlah: Nilai jumlah yang baru diinsert
  • NEW.idobat: ID obat yang baru diinsert
  • UPDATE obat SET stok = stok - NEW.jumlah: Kurangi stok obat
2

Cek Stok Obat SEBELUM (Baseline)

SELECT * FROM obat;

Catat stok sebelumnya (misal: Amoxillin = 20)

3

Insert Data ke Tabel TRANSAKSI_OBAT

Simulasi: Pasien beli Amoxillin 5 buah

INSERT INTO transaksiobat (idpasien, idobat, jumlah, totalharga) VALUES (1, 1, 5, 250);
4

Cek Stok Obat SESUDAH (Hasil Trigger)

SELECT * FROM obat WHERE idobat = 1;
  • Hasil yang seharusnya: Stok Amoxillin berubah dari 20 menjadi 15 (20-5)
  • Ini terjadi otomatis karena TRIGGER tgjual berjalan!
‚úÖ KESIMPULAN TRIGGER: TRIGGER mengotomatisasi logika business logic di database. Saat event terjadi (INSERT), trigger secara otomatis menjalankan perintah lain (UPDATE stok) tanpa perlu intervensi manual.

üìã Troubleshooting Replikasi

  • Slave_IO_Running = No: Cek koneksi network, user account, password

üîß Troubleshooting & FAQ

Solusi Masalah Umum

‚ùì

VM tidak mau start

Solusi:

  • Tunggu lebih lama (boot pertama bisa sampai 5 menit)
  • Pastikan VirtualBox sudah terinstall dengan benar
  • Cek hardware virtualization di BIOS (VT-x atau AMD-V)
  • Restart VirtualBox
‚ùì

Network ping tidak berjalan

Solusi:

  • Pastikan Host-Only Adapter sudah dikonfigurasi
  • Check firewall Windows (disable sementara untuk testing)
  • Restart network adapter di Ubuntu: sudo systemctl restart systemd-networkd
  • Buat Host-Only Network baru di VirtualBox: File ‚Üí Tools ‚Üí Network Manager
‚ùì

phpMyAdmin tidak bisa diakses

Solusi:

  • Pastikan Apache dan MySQL sudah aktif di XAMPP
  • Coba: http://localhost/phpmyadmin (di Ubuntu) atau http://127.0.0.1/phpmyadmin
  • Check port 80 tidak digunakan aplikasi lain
  • Restart XAMPP services
‚ùì

Replikasi database tidak berjalan

Solusi:

  • Cek Slave_IO_Running dan Slave_SQL_Running di phpMyAdmin replication status
  • Jika ada error, baca error message di "Show Slave Status"
  • Pastikan IP address master dan port 3306 benar
  • Check replication user account sudah dibuat di master
  • Restart services: STOP SLAVE; ‚Üí START SLAVE;
üí°
Tips Umum: Jika masalah tidak teratasi setelah coba solusi di atas, buat VM baru dari awal. Prosesnya hanya 30 menit dan lebih baik daripada troubleshoot berkepanjangan. Gunakan snapshot sebelum praktikum untuk backup.
