üåê Bagian 6: Konfigurasi Jaringan Host-Only

Setting Network untuk Replikasi Database

‚ö†Ô∏è
PENTING: Sesuai panduan UT resmi, gunakan Host-Only Adapter (192.168.56.x) untuk replikasi master-slave lokal. Ini lebih stabil dan sesuai dengan dokumentasi praktikum.
1

Buka Settings VM Ubuntu

  • Klik kanan pada VM "Ubuntu-Praktikum-2024"
  • Pilih "Settings"
  • Atau: Pilih VM ‚Üí Klik Settings di toolbar
2

Konfigurasi Network Adapter

  • Pilih menu "Network" di panel kiri
  • Tab "Adapter 1" harus sudah terbuka
  • Attached to: Pilih "Host-only Adapter"
  • Name: Pastikan "vboxnet0" terpilih
3

Advanced Settings (Optional)

  • Klik "Advanced" untuk memperluas opsi
  • Klik icon refresh (lingkaran hijau) di sebelah MAC Address
  • Ini untuk mendapatkan MAC address baru dan mencegah konflik
4

Klik OK untuk Menyimpan

üí° Tentang Host-Only Adapter

  • Hanya host (Windows 10) dan VM (Ubuntu) yang terhubung
  • IP range: 192.168.56.x (stabil dan tidak berubah)
  • Perfect untuk praktikum master-slave database lokal
  • Tidak perlu akses internet dari VM untuk replikasi bekerja
  • Jika VM perlu internet nanti, bisa tambahkan Adapter 2 dengan NAT

‚úÖ Bagian 10: Post-Installation Setup

Setup Pertama Setelah Instalasi Selesai

1

Tunggu Instalasi Selesai

  • Instalasi akan terus berjalan (copying files, configuring system)
  • Waktu: ~10-20 menit tergantung kecepatan
  • Setelah selesai, VM akan reboot otomatis
2

Login Screen Muncul

  • Klik username Anda dari list
  • Masukkan password yang Anda buat saat instalasi
  • Klik enter atau tombol login
3

First Boot & Welcome Screen

  • Ubuntu akan menampilkan welcome screen
  • Ada opsi untuk "Online Accounts" - skip (klik "Skip")
  • Ada opsi "Livepatch" - skip (klik "Next")
  • Ada opsi "Help Improve Ubuntu" - pilih sesuai preferensi
  • Klik "Done" untuk selesai
4

Install Guest Additions (IMPORTANT!)

  • Guest Additions meningkatkan performa dan user experience
  • Di menu VirtualBox VM, klik "Devices"
  • Pilih "Insert Guest Additions CD Image..."
  • Dialog akan muncul - klik "Run" atau buka Files dan double-click
  • Buka terminal di folder Guest Additions
cd /media/*/VBox* && sudo ./VBoxLinuxAdditions.run
  • Masukkan password Ubuntu Anda
  • Tunggu instalasi selesai (2-3 menit)
  • Reboot VM
5

Update System

  • Buka Terminal: Ctrl+Alt+T atau cari "Terminal" di search
  • Jalankan command:
sudo apt update && sudo apt upgrade -y
  • Masukkan password Anda
  • Ketik "y" jika diminta konfirmasi
  • Tunggu hingga selesai (5-15 menit)
  • Jika prompt "restart required" muncul, reboot VM

üîç Bagian 11: Verifikasi Network Connection

Test Koneksi Host-Only Network (SESUAI PANDUAN UT)

‚ö†Ô∏è
PENTING: Langkah ini sangat penting untuk memastikan replikasi database master-slave nanti berjalan dengan baik. Ikuti dengan teliti sesuai panduan UT resmi.
1

Cek IP Address di Windows 10 (Host)

  • Buka Command Prompt: Win+R ‚Üí ketik cmd
  • Jalankan command:
ipconfig
  • Cari section: "Ethernet adapter VirtualBox Host-Only Network"
  • Catat IP Address Windows Anda (biasanya 192.168.56.1)
2

Cek IP Address di Ubuntu (Guest)

  • Buka Terminal di Ubuntu: Ctrl+Alt+T
  • Jalankan command:
ip a

Atau:

ifconfig
  • Cari adapter yang dimulai dengan "192.168.56."
  • Catat IP Address Ubuntu Anda (misal: 192.168.56.104)
3

Test Koneksi dengan PING

Dari Ubuntu, ping Windows:

ping 192.168.56.1

Dari Windows, ping Ubuntu (Command Prompt):

ping 192.168.56.104
  • Jika hasilnya "Reply from..." ‚Üí BERHASIL ‚úÖ
  • Jika timeout atau "Host unreachable" ‚Üí ADA MASALAH ‚ùå
‚úÖ
JIKA BERHASIL: Lanjutkan ke praktikum transaksi database dan replikasi master-slave. Network sudah siap!
‚ùå
JIKA GAGAL: Check firewall Windows (mungkin block ping), atau restart network adapter di Ubuntu: sudo systemctl restart systemd-networkd

Tahapan Praktikum Database (T3-1 & T3-2)

Instalasi XAMPP di Windows 10 & Ubuntu

1

Windows 10: Download & Install XAMPP

  • Download dari: https://www.apachefriends.org/
  • Download versi Windows (apache, mysql, php)
  • Jalankan installer, ikuti langkah-langkah
  • Aktifkan Apache dan MySQL dari Control Panel XAMPP
  • Buka browser: localhost/phpmyadmin
2

Ubuntu: Install XAMPP via Terminal

# Download cd ~/Downloads wget https://www.apachefriends.org/xampp-linux-x64-installer.run # Buat executable chmod +x xampp-linux-x64-installer.run # Install (perlu sudo) sudo ./xampp-linux-x64-installer.run
  • Atau buka XAMPP Control Panel GUI dari Applications
  • Start Apache dan MySQL services
  • Akses phpMyAdmin: localhost/phpmyadmin

Praktikum Transaksi Database (T3-2)

  • 1. Create Database: Buat database "rumahsakit" dengan table pasien, dokter, etc (sesuai ERD T3-1)
  • 2. COMMIT: Membuat transaksi yang data insert-nya disimpan ke database
  • 3. ROLLBACK: Membuat transaksi yang data insert-nya dibatalkan
  • 4. SAVEPOINT/CHECKPOINT: Membuat transaksi dengan rollback ke checkpoint tertentu
  • 5. TRIGGER: Membuat trigger AFTER INSERT untuk update stok otomatis

üìñ Referensi: Buka PDF Panduan Prak T3-2 untuk detail SQL code dan langkah-langkah implementasi

Praktikum Replikasi Master-Slave (T3-2)

  • Master (Windows 10):
    • Konfigurasi my.ini dengan server-id dan binary logging
    • Buat replication user account
    • Export database rumahsakit ke file .sql
  • Slave (Ubuntu):
    • Import database rumahsakit dari master
    • Konfigurasi my.cnf dengan server-id dan replicate-do-db
    • Setup CHANGE MASTER TO dengan master hostname, user, password
    • START SLAVE untuk mulai replikasi
  • Testing: Insert/update data di master, lihat hasilnya langsung di slave

üîß Troubleshooting & FAQ

Solusi Masalah Umum

‚ùì

VM tidak mau start

Solusi:

  • Tunggu lebih lama (boot pertama bisa sampai 5 menit)
  • Pastikan VirtualBox sudah terinstall dengan benar
  • Cek hardware virtualization di BIOS (VT-x atau AMD-V)
  • Restart VirtualBox
‚ùì

Network ping tidak berjalan

Solusi:

  • Pastikan Host-Only Adapter sudah dikonfigurasi
  • Check firewall Windows (disable sementara untuk testing)
  • Restart network adapter di Ubuntu: sudo systemctl restart systemd-networkd
  • Buat Host-Only Network baru di VirtualBox: File ‚Üí Tools ‚Üí Network Manager
‚ùì

phpMyAdmin tidak bisa diakses

Solusi:

  • Pastikan Apache dan MySQL sudah aktif di XAMPP
  • Coba: http://localhost/phpmyadmin (di Ubuntu) atau http://127.0.0.1/phpmyadmin
  • Check port 80 tidak digunakan aplikasi lain
  • Restart XAMPP services
‚ùì

Replikasi database tidak berjalan

Solusi:

  • Cek Slave_IO_Running dan Slave_SQL_Running di phpMyAdmin replication status
  • Jika ada error, baca error message di "Show Slave Status"
  • Pastikan IP address master dan port 3306 benar
  • Check replication user account sudah dibuat di master
  • Restart services: STOP SLAVE; ‚Üí START SLAVE;
üí°
Tips Umum: Jika masalah tidak teratasi setelah coba solusi di atas, buat VM baru dari awal. Prosesnya hanya 30 menit dan lebih baik daripada troubleshoot berkepanjangan. Gunakan snapshot sebelum praktikum untuk backup.
